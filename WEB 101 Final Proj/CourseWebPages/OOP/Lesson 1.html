Lesson 1: Classes and Objects

In object-oriented programming (OOP), classes and objects are fundamental concepts that allow developers to model real-world entities and their interactions in code.

Classes

A class is a blueprint or template for creating objects. It defines the properties (attributes) and behaviors (methods) that all objects of that type will have.

Classes serve as a blueprint from which individual objects are created.

They encapsulate data and behavior, providing a way to organize and structure code.

Objects

An object is an instance of a class. It represents a specific, unique entity based on the blueprint defined by its class.

Objects have their own unique state (data) and behavior (methods), which are defined by the class.

Multiple objects can be created from the same class, each with its own set of data and behavior.

Example

Let's consider a simple example of a "Car" class

#include

#include

 

// Define the Car class

class Car {

public:

  // Attributes (data members)

  std::string brand;

  std::string model;

  int year;

 

 // Methods (member functions)

 void drive() {

  std::cout << "Driving the " << year << " " << brand << " " << model << ".\n" ;

}

 

 void stop() {

  std::cout << "Stopping the " << year << " " << brand << " " << model << ".\n" ;

 }

};

 

int main() {

 // Create objects of the Car class

 Car car1;

 Car car2;

 // Set data for car1

 car1.brand = "Toyota";

 car1.model = "Camry";

 car1.year = 2020;

 // Set data for car2

 car2.brand = "Ford";

 car2.model = "Mustang";

  car2.year = 2018;

 

 // Call methods on car1 and car2 objects

  car1.drive(); // Output: Driving the 2020 Toyota Camry.

  car2.stop(); // Output: Stopping the 2018 Ford Mustang.

 

return 0;

}

Explanation

In the example above, we define a class called "Car" with attributes (brand, model, year) and methods (drive, stop).

We then create two objects of the Car class: "car1" and "car2".

Each object has its own set of attributes (brand, model, year) and can call the methods (drive, stop) defined in the Car class.

Objects allow us to model individual cars with their own unique properties and behavior, while the class serves as a blueprint for creating and defining the structure of those objects.
