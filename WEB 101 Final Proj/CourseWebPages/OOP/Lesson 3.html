Lesson 3: Polymorphism and Abstraction

Polymorphism

Polymorphism is a key concept in object-oriented programming (OOP) that allows objects of different classes to be treated as objects of a common superclass. It enables a single interface to represent multiple forms or types. The term "polymorphism" is derived from the Greek words "poly," meaning "many," and "morph," meaning "form."

Polymorphism allows a single method or function to operate on different types of objects, providing flexibility and extensibility in software design. There are two main types of polymorphism: compile-time polymorphism (static polymorphism) and runtime polymorphism (dynamic polymorphism).

Compile-Time Polymorphism (Static Polymorphism)

Compile-time polymorphism occurs when the decision about which method to execute is made at compile time. It is achieved through method overloading and operator overloading.

Method Overloading

Method overloading involves defining multiple methods with the same name within the same class but with different parameter lists. The compiler determines which method to call based on the number and types of arguments passed to it.

class Calculator {

public:

  int add(int a, int b) {

   return a + b;

  }

  float add(float a, float b) {

   return a + b;

  }

};

Operator Overloading

Operator overloading allows operators such as +, -, *, /, etc., to be overloaded to work with user-defined data types. This enables the same operator to perform different operations depending on the operands' types.

class Complex {

private:

  double real;

  double imaginary;

public:

  Complex operator+(const Complex& other) {

   Complex result;

   result.real = this->real + other.real;

   result.imaginary = this->imaginary + other.imaginary;

   return result;

  }

};

Runtime Polymorphism (Dynamic Polymorphism)

Runtime polymorphism occurs when the decision about which method to execute is made at runtime, typically through method overriding and virtual functions.

Method Overriding

Method overriding involves defining a method in a subclass that is already present in the superclass. When a method is called on an object of the subclass, the overridden method in the subclass is executed instead of the superclass's method.

class Animal {

public:

  virtual void makeSound() {

   cout << "Some generic sound" << endl;

  }

};

class Dog : public Animal {

public:

  void makeSound() override {

   cout << "Woof!" << endl;

  }

};

Virtual Functions

Virtual functions are functions declared in a base class with the virtual keyword. They can be overridden in derived classes, and the appropriate function is called based on the object's actual type rather than its declared type.

class Shape {

private:

  virtual void draw() {

  cout << "Drawing a shape" << endl;

  }

};

class Circle : public Shape {

public:

  void draw() override {

  cout << "Drawing a circle" << endl;

  }

};

Benefits of Polymorphism

Code reusability: Polymorphism enables the reuse of methods and functions across different classes.

Flexibility: It allows for the creation of more generic and flexible code that can work with multiple types of objects.

Extensibility: New classes can be added without modifying existing code, promoting scalability and maintainability.

In summary, polymorphism is a powerful mechanism in OOP that facilitates code reuse, flexibility, and extensibility by allowing objects of different types to be treated uniformly.

Abstraction

Abstraction is a fundamental concept in computer science and programming that involves simplifying complex systems by focusing on essential characteristics while ignoring irrelevant details. It allows programmers to manage complexity by hiding implementation details and exposing only necessary functionalities to users.

Focus on Relevant Details

Abstraction involves identifying the essential features or behaviors of an entity or system and representing them in a simplified form. It helps in understanding the problem domain and designing solutions without getting bogged down by unnecessary details.

Hiding Implementation Details

Abstraction allows programmers to hide the internal workings of a system or component and provide a simplified interface for interaction. This separation of concerns enables modular design and reduces dependencies between different parts of a system.

Providing a Simplified Interface

Abstraction involves defining clear and concise interfaces that expose only the necessary functionalities to users or other components. This simplifies the usage of complex systems and promotes code readability, maintainability, and reusability.

Encapsulation of Complexity

Abstraction often goes hand in hand with encapsulation, where implementation details are encapsulated within well-defined modules or classes. This encapsulation shields users from the complexities of the underlying implementation, allowing them to focus on using the provided functionalities.
