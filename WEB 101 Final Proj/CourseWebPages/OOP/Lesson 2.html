Lesson 2: Encapsulation and Inheritance

Encapsulation

Encapsulation is one of the core principles of Object-Oriented Programming (OOP) that promotes data hiding and bundling data and methods within a class. It involves wrapping data (variables) and methods (functions) into a single unit called a class, and controlling access to that data through methods.

Data Hiding

Encapsulation hides the internal state of an object from the outside world. Only the methods of the class can directly access and modify the object's data, while external code interacts with the object through these methods.

This prevents unauthorized access to the object's internal state and protects it from being manipulated in unintended ways.

Bundling Data and Methods

Encapsulation bundles related data and methods together within a class, creating a self-contained unit. This promotes better organization and improves code maintainability by grouping related functionality.

By encapsulating data and methods within a class, developers can create reusable and modular components that can be easily integrated into different parts of the application.

Access Control

Encapsulation allows classes to define different levels of access to their members using access specifiers such as public, private, and protected.

Public members are accessible from outside the class, allowing external code to interact with the object.

Private members are only accessible within the class itself, preventing direct access from outside. Access to private members is typically provided through public methods, which can enforce validation and business rules.

Protected members are accessible within the class itself and its subclasses (derived classes), allowing for controlled access in inheritance hierarchies.

Example

#include

using namespace std;

 

class BankAccount {

private:

 string accountNumber; // Private member variable

 double balance; // Private member variable

 

public:

 // Public member methods

 void setAccountNumber(string accNum) {

  accountNumber = accNum;

 }

 void deposit(double amount) {

  if (amount > 0) {

   balance += amount;

    cout << "Deposit of $" << amount << " successful.\n" ; }

   else {

    cout << "Invalid deposit amount.\n" ;

   }

  }

  void displayBalance() {

   cout << "Account Balance: $" << balance << endl;

  }

};

intmain() {

 BankAccount myAccount;

 myAccount.setAccountNumber("123456789");

  myAccount.deposit(1000);

 myAccount.displayBalance();

return 0;

}

In this example:

The 'BankAccount' class encapsulates the account number and balance as private member variables.

Public member methods ('setAccountNumber', 'deposit', 'displayBalance') provide controlled access to the private data.

External code interacts with the 'BankAccount' object through these public methods, ensuring data integrity and security.

Benefits of Encapsulation:

Improved code maintainability and scalability.

Enhanced data security and protection from unauthorized access.

Simplified interface for interacting with objects, leading to cleaner and more readable code.

Promotion of code reuse through modular and self-contained components.

Inheritance

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a new class (subclass or derived class) to inherit properties and behaviors from an existing class (superclass or base class). This mechanism facilitates code reuse, promotes modularity, and enables the creation of hierarchical class structures.

Superclass and Subclass

Inheritance involves two classes: the superclass and the subclass.

The superclass is the existing class that provides the blueprint for creating objects.

The subclass is the new class that inherits properties and behaviors from the superclass.

Inheriting Properties and Behaviors

The subclass automatically gains access to all the members (fields, methods) of the superclass.

This includes both inherited attributes (fields) and inherited methods.

The subclass can extend the functionality of the superclass by adding new members or overriding existing ones.

Syntax of Inheritance

In most object-oriented languages like C++, Java, and Python, inheritance is implemented using the 'extends' (or ':,' in C++) keyword.

Syntax: 'class Subclass : public Superclass { ... }'

Types of Inheritance

Single Inheritance: A subclass inherits from only one superclass.

Multiple Inheritance: A subclass inherits from multiple superclasses (not supported in some languages like Java due to complexity and ambiguity).

Multilevel Inheritance: A subclass inherits from another subclass, forming a hierarchy.

Hierarchical Inheritance: Multiple subclasses inherit from the same superclass.

Example

#include

using namespace std;

 

class Animal {

public:

 void eat() {

  cout << "The animal is eating." << endl;

 }

 void sleep() {

  cout << "The animal is sleeping." << endl;

 }

};

// Subclass (Derived class)

class Dog : public Animal {

public:

 void bark() {

  cout << "The dog is barking." << endl;

 }

};

intmain() {

 // Create an object of the subclass (Dog)

 Dog myDog;

 

 // Access inherited methods from the superclass (Animal)

 myDog.eat(); // Output: The animal is eating.

  myDog.sleep(); // Output: The animal is sleeping.

 

 // Access methods specific to the subclass (Dog)

 myDog.bark(); // Output: The dog is barking.

 

return 0;

}

In this example:

the 'Dog' class inherits from the 'Animal' class. The 'Dog' class automatically gains access to the 'eat()'' and 'sleep()' methods from the 'Animal' class, while also defining its own method 'bark()'. This demonstrates how inheritance promotes code reuse and allows for the extension of functionality in subclasses.
